Background

What is an ECG?

💓
ECG Definition
Electrocardiogram — records the electrical activity of the heart over time as a waveform signal
📈
PQRST Waves
P wave → atria contract
QRS complex → ventricles contract (biggest peak!)
T wave → recovery phase
🏥
Clinical Use
Diagnose arrhythmias, heart attacks, and cardiac conditions in hospitals and wearables like Apple Watch

Our dataset — MIT-BIH — is single-lead ECG. Each heartbeat = 187 time-point values already normalised 0 to 1. Single-lead means 1 channel (not 12-lead hospital ECG).

Core Concept

Why LSTM for ECG?

❌ Simple RNN Problem
Forgets long-range patterns — can't connect what happened at time step 1 to time step 187. Called the vanishing gradient problem.
✅ LSTM Solution
Has a cell state (long-term memory) and hidden state (short-term memory). Three gates control what to remember and forget.
🗑️ Forget Gate
Decides what old information to throw away. Sigmoid output 0 = forget, 1 = keep
📥 Input Gate
Decides what new information to store. Sigmoid selects, tanh creates candidate values
📤 Output Gate
Decides what to output as hidden state. tanh of cell state × sigmoid filter

Data Preparation

Preprocessing Pipeline

Step 1 · Reshape to 3D
LSTM needs 3D input:

(87554, 187) → (87554, 187, 1)

samples × time steps × channels. Single-lead ECG = 1 channel only
Step 2 · One Hot Encoding
Labels converted to vectors:

0 → [1,0,0,0,0]
3 → [0,0,0,1,0]

Prevents model thinking class 4 is greater than class 0
Step 3 · Class Weights (Final)
Normal: 1x · Suprav: 5x
Ventric: 7x · Fusion: 15x
Unknown: 3x

Like giving extra attention to weaker students!
Already Normalised ✅
MIT-BIH data already 0–1 range. No MinMaxScaler needed. Zeros at end = padding to reach 187 time points for shorter beats.

Val loss jumping wildly — overshooting! 27x penalty too aggressive. Fusion F1 = 0.00 — never predicted at all.
Rejected · Overshooting problem
M2

Rejected · Stopped too early
M3
Adam · Manual weights {Normal:1, Fusion:5}
83% accuracy but Fusion F1 still 0.00. Weights not aggressive enough for rarest class with only 641 samples.

M4
AdamW · Aggressive weights Fusion:20x · LR=0.001
Crashed at epoch 3. Too high class weights + AdamW caused loss to spike uncontrollably.
Rejected · Crashed immediately
M5
AdamW · Balanced weights {Normal:1, Suprav:5, Ventric:7, Fusion:15, Unknown:3} · patience=5
Sweet spot found! 95.6% accuracy. Fusion F1 improved from 0.00 to 0.56. All rare classes now predicted correctly.

Quick Check

Test Your Understanding

Why did we use manual class weights instead of SMOTE for handling class imbalance in ECG data?
SMOTE creates synthetic heartbeats that don't represent real patients. In medical AI, fake ECG data is dangerous — a model trained on synthetic beats may fail on real patients. Manual class weights only change how the model learns from real data, keeping medical integrity intact.
What does overshooting look like in training graphs and how do we fix it?
Overshooting shows as val loss jumping up and down between epochs — like a bouncing ball! The learning rate is too high so the model takes steps that are too large and misses the minimum. Fix: reduce the learning rate (we went from 0.001 to 0.0005).
Lessons Learned

Key Challenges Faced

⚡ Overshooting
Learning rate too high → val loss bouncing up and down every epoch. We literally saw this in the loss plot! Fix: reduce LR from 0.001 to 0.0005.
⚖️ Class Weight Balance
27x weight → model panics and loses stability. 5x weight → rare classes still missed. 15x weight → the sweet spot! Took 7 iterations to find.
🛑 Early Stopping Timing
Patience=3 stopped training before rare classes could be learned. Increasing to patience=5 gave model enough time to improve on minority classes.
📊 Accuracy is Misleading!
83% accuracy model had Fusion F1 = 0.00! High accuracy hides failure on rare classes. Always check per-class F1 scores for imbalanced medical data.
Future Improvements

What Could Make This Better?

🔄
Bidirectional LSTM
Read ECG signal forwards AND backwards. Captures context from both directions — P wave before and T wave after the QRS complex gives richer understanding.
📉
Learning Rate Scheduler
Automatically reduce learning rate when val_loss plateaus — ReduceLROnPlateau callback. Would have fixed the unstable oscillating training we saw.
🎯
Focal Loss
Replaces categorical crossentropy. Automatically focuses on hard-to-classify rare samples. Better than manual class weights for extreme imbalance like Fusion (0.7%).
🔗
CNN + LSTM Hybrid
CNN extracts local waveform features (P, QRS, T shapes), LSTM captures temporal patterns across the full sequence. State-of-the-art approach for ECG.

🏥 Ultimate goal: 12-lead ECG (input shape 187×12) for full NHS clinical deployment — much richer cardiac data than single-lead wearables.
